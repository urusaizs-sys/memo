Community Plugin Analysis

Superpowers は
3層エンジニアリングの
実装パッケージである

Jesse Vincent（Prime Radiant）が開発したClaude Code用OSSプラグイン。14種類のスキルファイルをコンテキストに注入し、5フェーズの構造化ワークフローを強制することで、トークン消費を削減しながら出力品質を高める。その仕組みはPrompt / Context / Harnessの3層エンジニアリングそのものを体現している。

Key Insight

Superpowersは「プラグイン」という形でパッケージングされているが、本質的にはClaude Codeのコンテキストウィンドウに何をいつどの順序で注入するかを設計したContext Engineeringの産物。スキルファイル（markdown）がその媒体。

Skills
14スキルと層の対応

各スキルはmarkdownファイルとして配布され、セッション開始時にClaude Codeのコンテキストへ注入されます（Context Engineering）。同時に各スキル内に最適化されたプロンプト構造が記述されています（Prompt Engineering）。

🗺️ Plan Harness
タスクを2〜5分単位に分解。依存関係・エッジケース・順序を明示してから実行
🏛️ Architect Context
システム設計レベルの判断。コンポーネント間の関係・適用パターンを設計
🔬 TDD Harness
テスト駆動実行。並列タスク実行・コードレビュー・デバッグを構造化
🐛 Debug Harness
根本原因分析を強制。表面的な修正ではなく原因の特定から解決へ
🌿 Branch / Worktree Harness
git worktreeで作業を分離。フィーチャーブランチを自動作成・管理
🎨 Visual Companion Context
コーディング前にブラウザダッシュボードでモックアップ・レイアウト案を生成
🤖 Subagent Routing Harness
タスクを適切なサブエージェントへ自動振り分け。並列実行で高速化
💬 Clarify Prompt
実行前に曖昧な要件を質問形式で確認。スコープ漏れ・誤解を防止
📡 Dispatcher Harness
セッション開始時に自動起動。タスク種別を判定し適切なスキルへルーティング
📚 Docs Context
ライブドキュメント参照。設計フェーズ中に最新APIドキュメントを動的取得
🤝 Collaboration Harness
チーム開発フロー。レビュー・承認・共有ワークフローを構造化
⚡ Execute Prompt
プラン確定後の実行専用スキル。サブエージェントごとにタスクを割当
✅ Review Harness
2段階レビュー。実装直後と統合後の2回チェックポイントを強制
🧠 Brainstorm Prompt
仕様化前の発散フェーズ。アイデアを検証済みスペックへ昇華させる

Layer 1
Prompt Engineering として

「何もしないとClaude Codeはすぐコードを書き始める」という問題を、フェーズ別の最適プロンプト設計で解決します。

Superpowersなし vs あり — プロンプトの違い
❌ 素のClaude Code ``` User: “ユーザー認証機能を追加して” → Claude: すぐにコードを書き始める（方向が間違っていてもトークンを消費） ✅ Superpowers（Clarify スキル起動） User: “ユーザー認証機能を追加して” → Dispatcher: clarifyスキルをロード → Claude: ””” 実装前に確認させてください： 1. 認証方式は？（JWT / Session / OAuth） 1. 既存のユーザーテーブルはありますか？ 1. パスワードリセット機能は含みますか？ 1. 対象フレームワークは？（Next.js / Express…） “”” → 合意後にDesignフェーズへ進む
Layer 2
Context Engineering として

スキルファイル（markdown）がセッション開始時にコンテキストウィンドウへ注入されます。これは「何をモデルに見せるか」の設計そのものです。

CLAUDE.md — コンテキスト注入の設計
# CLAUDE.md（プロジェクトルートに配置） ## Superpowers Skills Loaded @skills/clarify.md # 要件確認プロセス @skills/architect.md # 設計パターン・原則 @skills/plan.md # タスク分解フォーマット @skills/tdd.md # テスト駆動チェックリスト @skills/dispatcher.md # セッション開始時ルーター …（計14スキル） ## Project Context - Stack: Next.js 14 / Prisma / PostgreSQL - Branch convention: feature/xxx, fix/xxx - Test runner: Vitest # → RAGなしで「制度知識」をモデルに与えるContext Engineering
```
Layer 3
Harness Engineering として

Superpowersが最も強力なのはHarness層です。フェーズゲート・状態管理・セッション復元・検証チェックポイントがすべて含まれています。

Harnessの機能 Superpowersでの実装 効果
フェーズゲート 各フェーズ完了まで次に進めない設計 方向誤りによるトークン浪費を防止
状態管理 plan.mdのチェックボックス セッション途中終了からの復元が可能
検証チェックポイント タスク完了後の2段階レビュー バグの早期発見・品質担保
ルーティング Dispatcherによる自動スキル選択 タスク種別に最適なワークフローへ誘導
リトライ制御 git worktreeによる作業分離 失敗時に安全にロールバック可能
Comparison
Superpowers vs Max Plan

同じ「トークン問題」への2つのアプローチ。削減するか、増やすか。

⚡ Superpowers Plugin
戦略無駄なトークンを削減
手段フェーズゲートで方向誤りを排除
コスト無料（OSSプラグイン）
向き複雑なフィーチャー開発
弱点クイックバグ修正には過剰
VS
☁️ Claude Max Plan
戦略使えるトークンを増量
手段上位プランで制限を緩和
コスト月額課金（有料）
向き大規模コードベース・長期タスク
弱点方向が間違えば消費も増大
推奨

両者は相補的です。Superpowersで無駄を排除しながら、大規模タスクにはMax Planの余裕を使う組み合わせが最も効率的。skip clarify プレフィックスでワンライナー修正時はフェーズをバイパス可能。
